# 요구사항: 대시보드

## 개요
어드민 첫 진입 화면. 현재 `/`는 로그인 후 바로 `/notices`로 리다이렉트됨.
서비스 핵심 지표를 한눈에 파악할 수 있는 홈 대시보드가 필요하다.

## 현재 상태
- `/` 경로: 라우터에서 `/notices`로 리다이렉트
- 집계 API 없음 → 백엔드 `/api/v1/admin/stats` 엔드포인트 추가 필요
- 사이드바에 "대시보드" 메뉴 없음

## 사용자 목표
- 어드민 진입 시 서비스 전체 상황을 즉시 파악
- 이상 지표(미답변 문의 급증, 회원 탈퇴 급증 등) 조기 발견

## 기능 요구사항

### 지표 카드 (상단 4개)
| 지표 | 설명 |
|------|------|
| 총 가입자 수 | 전체 유저 수 (탈퇴 포함) |
| 오늘 신규 가입 | 오늘 자정 이후 가입자 수 |
| 총 회고 수 | 저장 완료된 회고 누적 수 |
| 미답변 문의 수 | 답변 대기 중인 문의 수 |

### 최근 활동 (하단 2열)
- **최근 가입한 유저** (최대 5명): 이메일, 가입일, 직무, 클릭 시 `/users/:id`로 이동
- **최근 등록된 문의** (최대 5건): 제목, 유형, 등록일, 클릭 시 `/inquiries/:id`로 이동

### 네비게이션 변경
- 사이드바 최상단에 "대시보드" 메뉴 추가 (`/`)
- 라우터 `/` → `/notices` 리다이렉트 제거, 대신 대시보드 페이지 표시

## 비기능 요구사항
- 백엔드 API 1회 호출로 모든 지표 조회 (집계 API)
- 로딩 중 스켈레톤 UI
- 권한: ADMIN 이상

## 필요한 백엔드 작업
- `GET /api/v1/admin/stats` 엔드포인트 신규 추가
  ```json
  {
    "totalUsers": 1234,
    "newUsersToday": 12,
    "totalRetrospects": 5678,
    "unansweredInquiries": 3,
    "recentUsers": [...],
    "recentInquiries": [...]
  }
  ```
