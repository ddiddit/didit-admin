# 요구사항: 미션/레벨 관리

## 개요
v1.2.0에서 추가되는 레벨 시스템(Lv.1~10)과 미션을 어드민에서 조회/관리하는 화면.
현재 백엔드에 레벨/미션 시스템이 존재하지 않으므로 백엔드 구현 이후 진행.

## 현재 백엔드 상태
- 레벨/미션 관련 엔티티: **없음**
- 참고 가능한 기존 구조: 배지 조건 (`BadgeConditionType`), 회고 카운트 (`totalRetroCount`)

## v1.2.0 레벨 구조
| 레벨 | 미션 | 제한 |
|------|------|------|
| Lv.1 | 첫 회고 작성 | 없음 |
| Lv.2 | 7일 내 회고 3회 | 7일 만료 |
| Lv.3 | 2주 연속 주 1회 | 주차 초기화 |
| Lv.4 | 회고 3회 누적 | 없음 |
| Lv.5 | 3주 연속 주 1회 | 주차 초기화 |
| Lv.6 | 회고 5회 누적 | 없음 |
| Lv.7 | 4주 연속 주 1회 | 주차 초기화 |
| Lv.8 | 회고 7회 누적 | 없음 |
| Lv.9 | 5주 연속 주 1회 | 주차 초기화 |
| Lv.10 | 회고 10회 누적 | 없음 |

## 사용자 목표
- 레벨별 진행 중인 유저 수 파악
- 특정 레벨에서 이탈하는 유저 패턴 파악
- 운영 이슈 시 특정 유저 레벨 수동 조정

## 기능 요구사항

### 레벨 현황 대시보드 (`/missions`)
- 레벨별 유저 분포 (Lv.1~10, 레벨 미달성 포함)
  - 막대 차트 또는 테이블
- 이번 주 Lv.2 만료 위기 유저 수 (7일 내 미달성)
- 주간 레벨업 수

### 유저 레벨 조회
- 유저 목록 테이블 (레벨 컬럼 추가)
- 유저 검색 후 현재 레벨 + 미션 진행 상황 확인

### 유저 레벨 수동 조정 (SUPER_ADMIN)
- 특정 유저의 레벨을 강제로 변경 (운영 이슈 대응)
- 감사 로그 기록 필수

### 미션 알림 발송
- 특정 레벨(Lv.3/5/7/9) 진행 중이며 이번 주 미회고 유저에게 푸시 알림 발송
  - (현재 NotificationsPage의 타겟팅 기능 확장)

## 필요한 백엔드 작업 (대규모)
1. `user_levels` 테이블: userId, level, missionProgress, weeklyStreak, updatedAt
2. 레벨 달성 조건 체크 서비스 (`LevelConditionChecker`)
3. 회고 저장 이벤트 → 레벨업 체크 연동
4. 어드민 API:
   - `GET /api/v1/admin/missions/stats` — 레벨별 유저 분포
   - `GET /api/v1/admin/missions/users` — 유저별 레벨/미션 현황
   - `PATCH /api/v1/admin/missions/users/:userId/level` — 레벨 수동 조정
